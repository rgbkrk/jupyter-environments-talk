The Process

  • You write in the Notebooks, like always.
  • You (or we) use Cyrille Rossant‚Äôs excellent ipymd plugin to convert the content to Markdown. (Or, you can even just use Markdown as your native format.)
  • You check the Markdown version of the Notebooks into Atlas to collaborate with your editor, reviewers, and the O‚ÄôReilly production department.
  • You build a beautiful book.

Getting Beyond Books

  • Books are great, but they lack the features that makes the Notebook such an amazing experience: interactivity, immediacy, video, etc.
  • This is what learners (not just readers) expect

New Content Platform: Jupyter + Docker + tmpnb

  • Jupyter. Other than git and GitHub, we've never had authors demand a tool more than Jupyter.
  • Docker. Containerizing the notebooks gave us a simple way to package up and deploy the content and all its dependencies.
  • tmpnb. A lightweight system for creating ephemeral sandboxes.

Version 0.1: Just Enough Math

  • Hosted Notebooks on Docker+tmpnb for a Strata tutorial.
  • Great concept, but performance was an issue. (Static assets, etc)
  • Catastrophic failure mode. (No content at all!)

Thebe: Jupyter Cells As a Service

  • Jupyter code cells that can be embedded in static web pages.
  • Thebe. MIT-licensed plugin @zischwartz based on minrk/singlecell
  • Supports widgets, plots, and the wide range of Jupyter display protocols.
  • Separates serving of static assets from the notebook.
  • Much more graceful failure mode.
